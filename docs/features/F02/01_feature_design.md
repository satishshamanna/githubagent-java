# 01_feature_design.md: F02 - Pet Medical Records, Prescriptions & Vaccinations

This document describes the technical architecture and specifications for the pet medical records, prescriptions, and vaccinations feature in the Spring PetClinic application.

---

## 1. Feature Overview
To transition the application into a comprehensive clinical management system, we introduce clinical sub-records that link directly to visits. This enables veterinarians to record formal diagnoses, issue prescriptions, and log vaccinations with automated reminders for the pet owner.

### Core Flows:
1. **Clinical Diagnoses & Notes**: Vets can record specific diagnoses and severity levels during a pet's visit.
2. **Prescription Administration**: Vets can add prescription records (medication, dosage, and duration) linked to the pet and visit. Owners can view these active prescriptions on their pet details view.
3. **Vaccination Tracker**: Logs vaccines administered to pets, automatically calculating the next due date based on vaccine type, and showing active vaccination badges.

---

## 2. Database Schema (DDL)
We will introduce three new tables: `diagnoses`, `prescriptions`, and `vaccinations` to represent these clinical records.

### New Table: `diagnoses`
```sql
CREATE TABLE diagnoses (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    visit_id INT NOT NULL,
    description VARCHAR(255) NOT NULL,
    severity VARCHAR(20) DEFAULT 'NORMAL' NOT NULL, -- e.g., MILD, NORMAL, SEVERE, CRITICAL
    FOREIGN KEY (visit_id) REFERENCES visits(id)
);
```

### New Table: `prescriptions`
```sql
CREATE TABLE prescriptions (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    pet_id INT NOT NULL,
    visit_id INT NOT NULL,
    medication_name VARCHAR(100) NOT NULL,
    dosage VARCHAR(50) NOT NULL, -- e.g., "1 tablet", "5ml"
    frequency VARCHAR(50) NOT NULL, -- e.g., "Once daily", "Twice daily"
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status VARCHAR(20) DEFAULT 'ACTIVE' NOT NULL, -- e.g., ACTIVE, COMPLETED, DISCONTINUED
    FOREIGN KEY (pet_id) REFERENCES pets(id),
    FOREIGN KEY (visit_id) REFERENCES visits(id)
);
```

### New Table: `vaccinations`
```sql
CREATE TABLE vaccinations (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    pet_id INT NOT NULL,
    vaccine_name VARCHAR(100) NOT NULL,
    administered_date DATE NOT NULL,
    next_due_date DATE NOT NULL,
    status VARCHAR(20) DEFAULT 'COMPLETED' NOT NULL, -- e.g., COMPLETED, OVERDUE, UPCOMING
    FOREIGN KEY (pet_id) REFERENCES pets(id)
);
```

## 3. Backend Architecture & API Mappings

### Entity Mappings (Java)

```java
package org.springframework.samples.petclinic.model;

import java.time.LocalDate;
import jakarta.persistence.*;
import jakarta.validation.constraints.NotEmpty;
import jakarta.validation.constraints.NotNull;

@Entity
@Table(name = "prescriptions")
public class Prescription extends BaseEntity {

    @Column(name = "medication_name")
    @NotEmpty
    private String medicationName;

    @Column(name = "dosage")
    @NotEmpty
    private String dosage;

    @Column(name = "frequency")
    @NotEmpty
    private String frequency;

    @Column(name = "start_date")
    @NotNull
    private LocalDate startDate;

    @Column(name = "end_date")
    @NotNull
    private LocalDate endDate;

    @Column(name = "status")
    @NotEmpty
    private String status = "ACTIVE";

    @ManyToOne
    @JoinColumn(name = "pet_id")
    private Pet pet;

    @ManyToOne
    @JoinColumn(name = "visit_id")
    private Visit visit;

    // Getters and Setters...
}
```

### Repository Layer
Create interfaces for each new record type:
```java
package org.springframework.samples.petclinic.repository;

import java.util.List;
import org.springframework.samples.petclinic.model.Prescription;
import org.springframework.samples.petclinic.model.Vaccination;

public interface MedicalRecordRepository {
    void savePrescription(Prescription prescription);
    List<Prescription> findActivePrescriptionsByPetId(Integer petId);
    
    void saveVaccination(Vaccination vaccination);
    List<Vaccination> findVaccinationsByPetId(Integer petId);
}
```
