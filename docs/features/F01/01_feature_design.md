# 01_feature_design.md: F01 - Appointment Scheduling & Vet Availability

This document describes the technical architecture and specifications for the appointment scheduling and veterinarian availability features in the Spring PetClinic application.

---

## 1. Feature Overview
The appointment scheduler allows pet owners to request and book time slots with specific veterinarians based on their specialty. This shifts the application from a passive visit logging system to an active scheduling system.

### Core Flows:
1. **Owner Booking**: The pet owner selects a pet, filters veterinarians by specialty, selects a vet, views their available slots for a specific date, and submits a booking request.
2. **Veterinarian Dashboard**: A veterinarian can log in and view their daily, weekly, or monthly schedule of appointments, change appointment statuses (e.g., Scheduled, Completed, Cancelled), and log notes.

---

## 2. Database Schema (DDL)
We will introduce a new table `appointments` to represent booked slots.

### New Table: `appointments`
```sql
CREATE TABLE appointments (
    id INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    pet_id INT NOT NULL,
    vet_id INT NOT NULL,
    appointment_date DATE NOT NULL,
    time_slot VARCHAR(10) NOT NULL, -- e.g., "09:00", "10:30"
    description VARCHAR(255) NOT NULL,
    status VARCHAR(20) DEFAULT 'SCHEDULED' NOT NULL, -- e.g., SCHEDULED, COMPLETED, CANCELLED
    FOREIGN KEY (pet_id) REFERENCES pets(id),
    FOREIGN KEY (vet_id) REFERENCES vets(id),
    CONSTRAINT unique_vet_slot UNIQUE(vet_id, appointment_date, time_slot)
);
```

## 3. Backend Architecture & API Mappings

### Entity Mapping (Java)
We will introduce a new JPA entity `Appointment` mapping to the `appointments` table.

```java
package org.springframework.samples.petclinic.model;

import java.time.LocalDate;
import jakarta.persistence.*;
import jakarta.validation.constraints.NotEmpty;
import jakarta.validation.constraints.NotNull;

@Entity
@Table(name = "appointments", uniqueConstraints = {
    @UniqueConstraint(columnNames = {"vet_id", "appointment_date", "time_slot"})
})
public class Appointment extends BaseEntity {

    @Column(name = "appointment_date")
    @NotNull
    private LocalDate date;

    @Column(name = "time_slot")
    @NotEmpty
    private String timeSlot;

    @Column(name = "description")
    @NotEmpty
    private String description;

    @Column(name = "status")
    @NotEmpty
    private String status = "SCHEDULED";

    @ManyToOne
    @JoinColumn(name = "pet_id")
    private Pet pet;

    @ManyToOne
    @JoinColumn(name = "vet_id")
    private Vet vet;

    // Getters and Setters...
}
```

### Repository Layer
Create `AppointmentRepository` interface extending Spring Data JPA repository.
```java
package org.springframework.samples.petclinic.repository;

import java.time.LocalDate;
import java.util.List;
import org.springframework.data.repository.Repository;
import org.springframework.samples.petclinic.model.Appointment;

public interface AppointmentRepository extends Repository<Appointment, Integer> {
    void save(Appointment appointment) throws DataAccessException;
    List<Appointment> findByVetIdAndDate(Integer vetId, LocalDate date) throws DataAccessException;
    List<Appointment> findByPetId(Integer petId) throws DataAccessException;
    Appointment findById(int id) throws DataAccessException;
}
```

### Controller Layer
We will introduce `AppointmentController` with the following HTTP mapping endpoints:
- `GET /owners/{ownerId}/pets/{petId}/appointments/new`: Displays the appointment scheduling form.
- `POST /owners/{ownerId}/pets/{petId}/appointments/new`: Submits the new appointment booking.
- `GET /vets/{vetId}/appointments`: Displays the veterinarian appointment dashboard list.
- `POST /vets/{vetId}/appointments/{appointmentId}/status`: Updates appointment status.

## 4. Frontend View Design
We will add JSP templates under `src/main/webapp/WEB-INF/jsp/`:
1. `appointments/createOrUpdateAppointmentForm.jsp`:
   - Contains a form for selecting a `Vet` (grouped by specialty).
   - Date picker to select a date.
   - Dynamic time-slot selector (slots: 09:00, 10:00, 11:00, 13:00, 14:00, 15:00, 16:00) disabling already booked slots.
   - TextArea for appointment description.
2. `vets/vetDashboard.jsp`:
   - Lists appointments for a vet.
   - Allows marking them as completed or cancelled.

---

## 5. Architectural & Security Constraints
* **Double Booking Prevention**: A database-level unique constraint will prevent double booking of the same vet, date, and slot. The controller will validate this before saving and display a user-friendly error message if the slot is occupied.
* **Date Constraint**: Appointments must only be booked in the future (tomorrow or later).
* **Spring Transaction Manager**: `@Transactional` isolation will be added to the booking service layer method to avoid race conditions during concurrent bookings.
